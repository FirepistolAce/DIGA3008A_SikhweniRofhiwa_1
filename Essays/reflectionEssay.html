Reflection on My Website Implementation

This is the little corner in my website where I get to talk and give you a glimpse into my mind. This design journey has been nothing short of demanding if I may say but, are not all the things that matter. I cannot say there haven’t been times when I haven’t thought of giving up but, I had to remind myself that this is a journey and I'm only getting started.
I have to say that I have learned a lot about myself and my design process, I have learned that I am a very indecisive person and that I have to learn to trust my gut and go with my first instinct. I have also learned that I must be more open to feedback and criticism as it is a particularly important part of the design process. Another aspect I learnt was to be more patient with myself and my work as this is a process and it takes time to get it right.
This website project allowed me to apply what I have learned this semester and reflect on how my UI (User Interface) and UX (User Experience) design choices shaped the structure and coding of my site.

Folder Structure: Organization for Scalability and Maintainability

A well-organized folder structure is the foundation of any efficient web development project. From the very beginning, I prioritized clarity and logical separation to ensure that my project would be easy to navigate, modify, and scale in the future. I divided my files into distinct folders:
• HTML – Containing all markup files • CSS – Housing stylesheets • JavaScript – Storing scripts for interactivity • Images – Managing all visual assets
From the beginning, I prioritized an organized folder structure with a clear hierarchy. I separated my files into distinct folders for HTML, CSS, JavaScript, and images, following the principle of separation of concerns. This approach kept the presentation, logic, and content layers meticulously organized, making navigation and updates more efficient. Consistent naming conventions also ensured scalability, so future expansions or modifications will be straightforward. It also ensured that each aspect of the project had its own dedicated space, making debugging and updates significantly easier. For example, if I needed to adjust a styling rule, I knew exactly where to find the relevant CSS file without sifting through unrelated code.
Additionally, I maintained consistent naming conventions (e.g., header.css for header-related styles, script.js for main functions) to avoid confusion. This practice not only helped me during development but would also make it easier for another developer to understand my project structure if they were to collaborate on it later. One challenge I faced was deciding whether to further break down CSS or JavaScript files into smaller modules (e.g., separate files for animations, layouts, or components). While I initially considered this, I opted for a single CSS and JS file for this project due to its small scale. However, for larger applications, I now recognize the benefit of modularizing styles and scripts even further to improve maintainability.

Metadata: Optimizing for SEO and Cross-Browser Consistency

Metadata might seem like a minor detail, but it plays a crucial role in how browsers and search engines interpret a website. In the section of each HTML file, I included:
• Viewport settings (meta name="viewport" content="width=device-width, initial-scale=1.0">) to ensure responsive behaviour on mobile devices.
• Character encoding (meta charset="UTF-8">) to handle special characters and symbols correctly.
• Page-specific titles and descriptions to improve SEO and user experience when the site appears in search results.
By including these elements, I ensured that my website would load correctly across different browsers and devices, providing a consistent user experience. The viewport settings were particularly important for mobile responsiveness, allowing the site to adapt to various screen sizes without losing functionality or readability. The character encoding ensured that all text displayed correctly, regardless of the characters used, which is essential for internationalization and accessibility. Page-specific titles and descriptions not only improved SEO but also helped users understand the content of each page at a glance, enhancing navigation and usability. I included essential metadata in the (head) section of each HTML file to optimize browser interpretation and responsiveness. Proper viewport settings, character encoding, and page-specific titles and descriptions improved SEO and ensured consistent rendering across devices. This diligence helps the site load correctly and appear appropriately in search results.

CSS was one of the most challenging yet rewarding aspects of this project. My primary goals were:

1. Avoiding inline styles – Keeping all styling in an external stylesheet for consistency and easier updates.
2. Responsive design – Ensuring the site looked good on all devices using media queries.
3. Modularity – Grouping related styles together (e.g., navigation styles in one section, typography in another).
To maintain consistency, I avoided inline styles and relied on an external CSS file. This allowed me to reuse styles across pages and make global changes efficiently. I organized my CSS with logical class and ID naming, grouping related styles for better readability. The design is fully responsive, using media queries to adapt layouts for mobile, tablet, and desktop screens. Keeping the CSS clean and modular ensures easy maintenance for future updates. Not to mention each of my styling files have a descriptive name that goes hand in hand with the task that they perform to not get confused when changing or fixing code.

JavaScript: Balancing Interactivity and Performance

JavaScript brought my website to life by adding dynamic features such as:
• An interactable glowing feature box in the home screen.
• Scroll back to top button.
• Navigation buttons in between blogs.
I avoided inline scripts and instead used an external JavaScript file with event listeners (click, scroll) and DOM manipulation to keep functionality separate from markup.
I used JavaScript to enhance interactivity, such as grabbing the user’s attention both in the load screen and the home screen with a quote box and an interactable feature glowing box. Event listeners and DOM manipulation ensured smooth user interactions. All scripts were placed in a separate file, and reusable functions (like class back up button) were kept modular for better performance and readability.

Conclusion

Overall, my website’s structure and functionality were guided by UI/UX principles. A modular, accessible, and user-friendly approach not only improved the experience for visitors but also made the codebase more organized and scalable. This project deepened my understanding of how front-end and back-end decisions impact the final product, reinforcing the importance of thoughtful development practices. Not forgetting the importance of planning out what you want to do from the beginning, rather than that freestyling as time goes.
